# Render 云部署完整教程

将 Python HTTP 服务免费部署到 Render.com，手机随时随地访问。

---

## 一、项目准备

### 1.1 确保项目结构规范

```
项目目录/
├── .gitignore              # 排除缓存、数据文件、IDE 配置
├── README.md               # 项目说明
├── requirements.txt        # Python 依赖（仅运行时需要的）
├── 主程序.py               # HTTP 服务入口
├── 前端页面.html           # 看板/网页
└── 其他资源文件
```

### 1.2 requirements.txt 只写运行时依赖

```
akshare>=1.18.0
pandas>=2.0.0
```

> ⚠️ 不要包含 `websockets`、`qrcode`、`Pillow` 等开发期或已废弃的依赖。

### 1.3 .gitignore 排除不需要上传的文件

```gitignore
__pycache__/
*.pyc
history_data/       # 本地生成的运行时数据
.claude/
.vscode/
Thumbs.db
.DS_Store
```

### 1.4 服务端口适配云环境

```python
PORT = int(os.environ.get("PORT", 8765))
```

Render 通过 `PORT` 环境变量分配端口，本地运行时默认用 8765。

## 二、推送到 GitHub

### 遇到的问题及解决

#### 问题 1：`remote origin already exists`

```bash
# 查看已有远程
git remote -v

# 如果地址不对，更新
git remote set-url origin https://github.com/用户名/仓库名.git
```

#### 问题 2：`Failed to connect to github.com port 443`

原因：Git 默认的 `openssl` SSL 后端在某些 Windows 网络环境（含代理/VPN）下无法建立 HTTPS 连接。

解决：切换为 Windows 原生 `schannel` SSL 后端。

```bash
# 切换到 Windows 原生 SSL（一劳永逸）
git config --global http.sslBackend schannel

# 如需切回 openssl
# git config --global http.sslBackend openssl
```

#### 问题 3：`Permission denied (publickey)`

原因：未配置 SSH 密钥。直接用 HTTPS 方式推送即可，无需 SSH。

#### 问题 4：首次连接 `SSL certificate verification has been disabled` 警告

检查是否误关了 SSL 验证：

```bash
# 查看全局配置
git config --global --list | grep ssl

# 如有 http.sslverify=false，恢复
git config --global http.sslverify true
```

### 完整推送命令

```bash
# 1. 初始化（如果还没有）
cd 项目目录
git init

# 2. 配置用户信息（仅首次）
git config user.email "你的邮箱"
git config user.name "你的用户名"

# 3. 添加文件
git add .

# 4. 提交
git commit -m "init: 项目描述"

# 5. 关联远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 6. 推送
git push -u origin master
```

## 三、Render 部署

### 3.1 登录 Render

打开 https://render.com → 用 GitHub 账号登录（授权访问仓库）。

### 3.2 创建 Web Service

1. 点右上角 **New +** → **Web Service**
2. 选择你的 GitHub 仓库
3. 填写配置：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **Name** | 自定义（如 `gold-dashboard`） | 自动生成 URL |
| **Region** | 默认即可 | |
| **Branch** | `master` | |
| **Runtime** | `Python 3` | |
| **Build Command** | `pip install -r requirements.txt` | 安装依赖 |
| **Start Command** | `python gold_server.py` | 启动服务 |
| **Instance Type** | **Free** | 免费方案 |

4. 点 **Deploy Web Service**

### 3.3 等待部署

- 构建 + 首次启动约 3-5 分钟
- 日志显示 `[READY] 服务已启动` 即完成
- 页面顶部会显示你的 URL，如 `https://gold-dashboard-c005.onrender.com`

## 四、使用与注意事项

### 免费版限制

| 限制 | 说明 |
|------|------|
| 每月 750 小时 | 个人全天候使用完全足够 |
| 15 分钟休眠 | 无人访问 15 分钟后自动休眠，下次访问需等 30-60 秒冷启动 |
| 带宽 100GB/月 | 个人看板使用绰绰有余 |
| 磁盘空间 | 数据存在云端，不会丢失 |

### 手机使用

- 浏览器直接打开 `https://你的地址.onrender.com`
- 支持 HTTPS，可添加到手机主屏幕
- 任何网络（4G/5G/WiFi）都能访问

### 更新代码

```bash
# 本地修改后，推送即可自动重新部署
git add .
git commit -m "更新说明"
git push
```

Render 会自动检测 GitHub 变更并重新部署。

## 五、踩坑总结

| 问题 | 现象 | 根因 | 解决 |
|------|------|------|------|
| Git 无法连接 GitHub | `Failed to connect to github.com port 443` | Windows 上 `openssl` SSL 后端与网络环境不兼容 | `git config --global http.sslBackend schannel` |
| SSH 推送被拒 | `Permission denied (publickey)` | 未配置 SSH 密钥 | 用 HTTPS 地址推送 |
| requirements.txt 过大 | 安装慢、构建失败 | 包含了非运行时依赖 | 只保留 `akshare` + `pandas` |
| PWA 安装失败 | 手机"无法安装此应用" | HTTP 访问不支持 PWA（仅 localhost 例外） | Render 提供 HTTPS，可正常安装 |
| 首次访问超时 | 页面加载很慢 | 免费版有冷启动 | 等待 30-60 秒后刷新 |
