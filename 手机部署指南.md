## 三、Render.com 部署配置

### 创建 Web Service

| 配置项 | 填写的值 |
|--------|----------|
| Name | `gold-dashboard` |
| Runtime | `Python 3` |
| Build Command | `pip install -r requirements.txt` |
| Start Command | `python gold_server.py` |
| Instance Type | **Free** |

### 部署流程

1. 打开 https://render.com → GitHub 登录
2. **New +** → **Web Service** → 选择仓库
3. 填入上表配置 → **Deploy Web Service**
4. 等待 3-5 分钟 → 获得 `https://xxx.onrender.com` 地址

## 四、遇到的问题及解决

| # | 问题 | 原因 | 解决 |
|---|------|------|------|
| 1 | `git push` 报 `Failed to connect to github.com port 443` | Windows 上 Git 默认 `openssl` SSL 后端与代理/VPN 不兼容 | `git config --global http.sslBackend schannel` |
| 2 | `git push` 报 `Permission denied (publickey)` | 未配置 SSH 密钥 | 用 HTTPS 远程地址，不用 SSH |
| 3 | `ssl certificate verification has been disabled` 警告 | 之前尝试过程中误关了 SSL 验证 | `git config --global http.sslverify true` |
| 4 | 手机扫码无二维码显示 | qrcodejs CDN (`cdn.jsdelivr.net`) 国内加载慢 | 已保留 CDN 方案；可改为服务端 `/api/qrcode` 生成 |
| 5 | 手机"添加到主屏幕"报无法安装 | HTTP 不支持 PWA 安装（仅 localhost 例外） | Render 提供 HTTPS，可正常安装 PWA |
| 6 | Render 首次访问很慢 | 免费版 15 分钟无访问自动休眠 | 等待 30-60 秒冷启动后正常使用 |
| 7 | akshare 东方财富接口代理错误 | `fund_etf_hist_em` 走东方财富，被代理拦截 | 改用新浪源 `fund_etf_hist_sina(symbol="sh518880")` |
| 8 | `futures_foreign_realtime` / `stock_zh_a_realtime_em` 不存在 | akshare 版本更新，旧 API 已改名 | 改用 `futures_foreign_commodity_realtime` / `fund_etf_spot_em` |

## 五、本地 vs 云端对比

| | 本地模式 | 云端模式 |
|------|----------|----------|
| 启动方式 | 双击 `启动看板.bat` | 自动运行 |
| 访问地址 | `http://localhost:8765` | `https://xxx.onrender.com` |
| 手机条件 | 同一 WiFi | 任何网络 |
| HTTPS | ❌ | ✅ |
| PWA 安装 | ❌ | ✅ |
| 二维码 | 局域网地址 | 不需要（直接开 URL） |
| 费用 | 免费 | 免费（750h/月） |
| 适用场景 | 家里电脑开着看 | 随时随地手机查看 |
